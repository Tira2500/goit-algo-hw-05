# Порівняльний аналіз алгоритмів пошуку підрядка

## Алгоритми

- **КМП (Кнута-Морріса-Пратта)** — використовує префікс-функцію щоб пропускати вже перевірені позиції. Складність: O(n+m).
- **Боєр-Мур** — порівнює з кінця шаблону і робить великі стрибки завдяки таблиці зсувів. Складність: O(n/m) у кращому випадку.
- **Рабін-Карп** — використовує хешування для порівняння підрядків. Складність: O(n+m) у середньому.

## Аналіз результатів

### Боєр-Мур

Найшвидший алгоритм у всіх тестах. Завдяки таблиці зсувів робить великі
стрибки при невідповідності — особливо ефективний на вигаданих підрядках
де може пропускати багато символів одразу.

### КМП

Показує стабільні результати. На існуючих підрядках близький до Боєра-Мура,
але на вигаданих помітно повільніший — перевіряє більше позицій.

### Рабін-Карп

Найповільніший у всіх тестах через накладні витрати на обчислення хешів.
Особливо програє на вигаданих підрядках де проходить весь текст до кінця.

---

## Висновки

1. **Боєр-Мур — переможець** в обох текстах і для обох типів підрядків.
   Таблиця зсувів дозволяє пропускати великі частини тексту що робить
   його найефективнішим на практиці.

2. **КМП** — надійний вибір коли важлива передбачувана складність O(n+m).
   Особливо ефективний коли підрядок знаходиться на початку тексту.

3. **Рабін-Карп** програє через витрати на хешування. Його перевага
   проявляється при пошуку **кількох підрядків одночасно** — тут він
   може бути ефективнішим за інших.

4. **Загальний висновок**: для пошуку одного підрядка у великому тексті
   найкращим вибором є **алгоритм Боєра-Мура**.
